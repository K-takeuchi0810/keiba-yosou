# JV-Link リターンコード詳細

`docs/JV-Link4901.pdf` p53-56 の「3. コード表」より。

## メソッド別の主要返り値

### `JVInit(sid)`
| rc | 意味 | 対処 |
|---|---|---|
| 0 | 正常 | — |
| -101 | sid が設定されていない | sid パラメータを正しく渡す |
| -102 | sid が 64 バイト超過 | 64 バイト以内に |
| -103 | sid 不正 (1 桁目がスペース) | スペース以外の文字で開始 |

### `JVOpen` / `JVRTOpen`
| rc | 意味 | 対処 |
|---|---|---|
| 0 以上 | 正常 (4 要素タプル: rc, readcount, downloadcount, last_timestamp) | — |
| -1 | 該当データ無し | 新着なし。`JVClose()` で終了 |
| -2 | セットアップダイアログでキャンセル | `JVClose()` で終了 |
| -111 | dataspec 不正 | dataspec 文字列を確認 |
| -112 | fromtime 不正 (開始時刻) | yyyymmddHHMMSS 14 桁か確認 |
| -113 | fromtime 不正 (終了時刻) | `-` 以降の終了時刻を確認 |
| -114 | key 不正 (JVRTOpen) | レース識別キーを確認 |
| -115 | option 不正 | option=1/2/3/4 のいずれかか確認 |
| -116 | dataspec と option の組み合わせが不正 | 組み合わせ表を参照 |
| -201 | JVInit 未呼び出し | 先に `JVInit(sid)` |
| -202 | 前回 Open に対し JVClose 未呼び出し | 冒頭で `JVClose()` を呼ぶ |
| -211 | レジストリ内容が不正 | JV-Link 再インストール |
| -301 | 認証エラー (利用キー不正/重複) | キー再発行 |
| -302 | 利用キー有効期限切れ | サービス再購入 |
| -303 | 利用キー未設定 | `JVSetUIProperties` で登録 |
| -305 | 利用規約未同意 | JV-Link 本体で同意 |
| -401 | JV-Link 内部エラー | JRA-VAN 連絡 |
| -411 | サーバエラー (HTTP 404) | サーバ問題 |
| -412 | サーバエラー (HTTP 403) | サーバ問題 |
| -413 | サーバエラー (HTTP その他) | サーバ問題 |
| -421 | サーバ応答不正 | サーバ問題 |
| -431 | サーバアプリ内部エラー | サーバ問題 |
| -501 | スタートキット (CD/DVD) 無効 | 提供終了 (2022/3) |
| -504 | サーバメンテナンス中 | 待つ |

### `JVStatus`
| rc | 意味 |
|---|---|
| 0 以上 | ダウンロード済ファイル数 |
| -201 | JVInit 未呼び出し |
| -203 | JVOpen 未呼び出し |
| -502 | ダウンロード失敗 |

### `JVRead` / `JVGets`
| rc | 意味 | 対処 |
|---|---|---|
| 0 以上 | バッファに格納したバイト数 | データを処理 |
| 0 | 全ファイル読み込み完了 (EOF) | ループ終了 |
| -1 | ファイル切替 | エラーではない。次ファイルへ |
| -3 | ファイルダウンロード中 | 少し待って再試行 |
| -201 | JVInit 未呼び出し | — |
| -202 | JVClose 未呼び出し (前回 Open 残り) | — |
| -203 | JVOpen 未呼び出し | — |
| -402 | DL ファイル異常 (サイズ 0) | `JVFiledelete` 後 JVOpen やり直し |
| -403 | DL ファイル異常 (内容) | 同上 |
| -502 | ダウンロード失敗 | 原因除去後 JVOpen やり直し |
| -503 | ファイル見つからない | JVOpen やり直し |

### `JVClose`
| rc | 意味 |
|---|---|
| 0 | 正常 |

(エラーが返ることはほぼない。例外で握り潰して可)

### `JVFiledelete`
| rc | 意味 |
|---|---|
| 0 | 正常 |
| その他 | ファイルが存在しない等 (実害なし) |

## 既存実装でのハンドリング

[jvlink_client/client.py](jvlink_client/client.py) では:
- `-1` (ファイル切替): `current_handle.close()` してハンドル切替
- `-3` (DL 中): `time.sleep(0.5)` で再試行
- `-402 / -403` (異常ファイル): `JVFiledelete` 後その dataspec の取得を打ち切り
- 上記以外の `rc < 0`: `JVLinkError` を raise

警告系のリカバリは `on_progress("warn", {...})` で GUI に通知。

## ハマりやすい挙動

### `-1` は 2 通りある
- `JVOpen` の `-1`: 該当データなし (新着なし)。エラー終了
- `JVRead` の `-1`: ファイル切替 (正常)。次ファイルへ続行

メソッドで意味が変わる。

### `JVStatus` が `downloadcount` に到達しないまま `JVRead` を呼ぶと `-3` ループに入る
正解は: `downloadcount > 0` のときだけ `JVStatus` で全 DL を待ってから `JVRead`。`downloadcount == 0` はキャッシュにある (新規 DL 不要) ので即 `JVRead` に進む。

### `JVRead` が `0` を返す前に `-1` が出る
これは正常。1 ファイル読み終わるたびに `-1` が来る。`0` (EOF) になるのは全ファイル読み終わったとき。
