# JV-Link dataspec 一覧

`JVOpen(dataspec, ...)` / `JVRTOpen(dataspec, ...)` で指定する dataspec 文字列と、それぞれで取得できるレコード種別の対応。

## 蓄積系 (option=1, 通常差分)

| dataspec | 内容 | 主なレコード |
|---|---|---|
| `TOKU` | 特別登録馬 | `TK` |
| `RACE` | 競走情報 (レース・出走馬・払戻 等) | `RA` `SE` `HR` `H1` `H6` `O1`〜`O6` |
| `DIFF` | 開催情報差分 | `RA` `SE` 等の差分 |
| `BLOD` | 血統情報 | `BT` (系統) `HN` (繁殖) `SK` (産駒) |
| `SNAP` | 出走時点情報 | `SE` のスナップ |
| `SLOP` | 坂路調教 | `HC` |
| `WOOD` | ウッドチップ調教 | `WC` |
| `YSCH` | 年間スケジュール | `YS` |
| `HOSE` | 競走馬基本 | `UM` |
| `HOYU` | 馬主基本 | `BN` |
| `DIFN` | 開催情報差分 (今週) | RACE と類似 |
| `BLDN` | 血統情報 (今週) | BLOD と類似 |
| `SNPN` | 出走時点情報 (今週) | SNAP と類似 |
| `HOSN` | 競走馬基本 (今週) | HOSE と類似 |

## セットアップ系 (option=3 / 4)

option=1 のセット + 以下を追加:

| dataspec | 内容 |
|---|---|
| `COMM` | コメント (記者コメント等) |
| `MING` | データマイニング予想 |

## 速報系 (JVRTOpen で取得)

key の指定方法は dataspec ごとに異なる。

| dataspec | 内容 | key 形式 |
|---|---|---|
| `0B12` | 速報 馬体重 | レース識別 (年月日場回日R = 14 桁) |
| `0B14` | 速報 開催情報 (一括) | 年月日 (8 桁) |
| `0B15` | 速報 票数 | レース識別 |
| `0B30` | 速報 払戻 | レース識別 |
| `0B11` | 速報 オッズ | レース識別 |
| `0B16` | 速報 開催情報 (指定) | レース識別 |
| `0B20` | 速報 タイム型データマイニング | レース識別 |
| `0B41` | 速報 重勝式オッズ | yyyymmdd |

key の正確な仕様は `docs/JV-Link4901.pdf` の各メソッド説明を参照。

## どの dataspec を選ぶか (実用視点)

### 当日の予想に必要 (速報 / RT)
- `0B12` 馬体重
- `0B11` 速報オッズ (`O1` の発走前更新)
- `0B14` / `0B16` 出馬表確定情報

### 過去データの本格セットアップ
- option=4 で `RACE` `BLOD` `HOSE` `HOYU` `MING` を一気に取得
- 数 GB / 数十分かかる。初回のみ。

### 普段の差分更新
- option=1 で `RACE` `DIFN` `BLDN` `HOSN` を週次 (火曜・木曜) で

### 予想ロジック強化
- `BLOD` (血統) → 産駒成績との関連
- `SLOP` `WOOD` (調教) → 直前の動き
- `MING` (マイニング) → JRA-VAN 公式予想スコア
