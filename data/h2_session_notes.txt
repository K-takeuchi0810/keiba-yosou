「弱く / 強く / 確定」判定強度 rubric (v4 新規、H1 反省から)
================================================================

H1 v3 では「弱く棄却」「弱く採用」「border zone」「weakly clear」を
直感で書き分けた。v4 では下記 rubric で機械的に判定する。

統計的検定 (permutation / bootstrap / t-test 等) の場合:

| 強度ラベル | 判定基準 |
|---|---|
| 確定的に棄却 | p < 0.01 |
| 強く棄却 | 0.01 ≤ p < 0.05 |
| 弱く棄却 | 0.05 ≤ p < 0.10 |
| border zone | 0.10 ≤ p < 0.20 |
| 弱く支持 (= 棄却失敗 + 観察方向一致) | 0.20 ≤ p < 0.50 (片側) |
| 判定不能 | sample size 不足 (N が p 計算自体を疑わせる) |
| 強く支持 | 同 p < 0.05 で逆方向 |
| 確定的に支持 | p < 0.01 で逆方向 |

観察的判定 (fold 数、戦略数、bets 数等) の場合:

| 強度ラベル | 判定基準 |
|---|---|
| 確定的に棄却 | 全 fold で 0 件 / 全 fold で反例 |
| 強く棄却 | 全 fold で過半数反例 |
| 弱く棄却 | 過半数 fold で反例 (= 2/3 以上) |
| border zone | 半々 (1/3 で反例、2/3 で支持 等) |
| 弱く支持 | 過半数 fold で支持 (= 2/3 以上) |
| 強く支持 | 全 fold で支持 |
| 確定的に支持 | 全 fold + 別 dataset で支持 |

各判定で「rubric のどこに該当するか」を明示して書く。
直感で「弱く」を書かない。

§5 Phase B1 着手判定 ALL/SOME/NO-PASS rubric (v4 新規)
================================================================

v3 §5 では「weakly clear」「partially clear」「not strongly clear」と
intuition で書いた。v4 では下記 3 条件をそれぞれ binary (PASS / NO) で
判定し、合計で ALL-PASS / SOME-PASS (2/3) / NO-PASS (≤ 1/3) を出す。

条件 1 (H1 完了 = U3, U4, U6 が「判定不能」を脱した)
  PASS if: U3, U4, U6 のうち 3 つすべてに「弱く / 強く / 確定」のいずれか
           の rubric ラベルが付いている
  NO    if: 1 つでも「判定不能」のまま

条件 2 (採用判定基準 0.80/0.50 が現実的に achievable)
  PASS if: (a) N7 で R%=80 cliff edge の対立仮説 (a)(b)(c) のいずれかが
           rubric で「弱く支持」以上 かつ
           (b) N6 で favorite class achievable が「弱く支持」以上
  NO    if: 上記 (a)(b) のいずれかが border zone 以下

条件 3 (規範外バイアス: season + fold selection が H1+H2 で否定)
  PASS if: N3 (season) で R1 棄却が rubric で「弱く棄却」以上 かつ
           N8 (fold) で fold selection bias が rubric で「弱く棄却」以上
  NO    if: いずれかが border zone 以上残置

総合判定:
  ALL-PASS (3/3): Phase B1 plan 書き直し着手可
  SOME-PASS (2/3): 部分着手可 (PASS した条件を前提に、NO 条件は plan で
                   独立に追加対策)
  NO-PASS (≤ 1/3): 着手不可 (H1+H2 でも基準解けていない)

各 N の判定基準 (handoff §H2 + 上記 rubric、結果を見る前に固定)
================================================================

N1 (月次 LGBM Brier):
  入力: scripts/dump_predictions.py の per-pick CSV (2025-01〜2026-05)
        per-pick = filter_sweep の Pick + actual_winner ラベル
  計算: 月次に Brier = mean[(p_predicted - y_actual)^2] for top-pick horses
        + reliability: prob bin (0-10%, 10-20%, ...) ごとの hit_rate
  事前予想 (確証バイアス対策): 2026 月次 Brier は 2025 平均 ± 1σ 内、
                                drift は「弱く棄却」レンジ
  判定基準:
    - 2026 月次平均 Brier > 2025 月次平均 + 2σ: drift 強く支持 (U1-b 採用)
    - 2026 月次平均 Brier > 2025 月次平均 + 1σ: drift 弱く支持
    - 2026 月次平均 Brier ∈ 2025 平均 ± 1σ: drift 弱く棄却
    - 2026 月次平均 Brier < 2025 平均 - 1σ: drift 強く棄却 (LGBM 改善)
    - サンプル数不足 (< 100 picks/月): 判定不能

N8 (fold 分割 sensitivity):
  入力: scripts/dump_predictions.py の per-pick CSV
  計算: 同じ pick データを 4 種類の fold 分割で再集計、dm_rank_1_3 戦略の
        robust=Y 件数を見る
  fold 分割案:
    (i)   現状 = 2025H1 / 2025H2 / 2026P
    (ii)  四半期 = 2025Q1-Q4 / 2026Q1-Q2 (6 fold)
    (iii) 逆順 = 2026P / 2025H2 / 2025H1 (順序のみ)
    (iv)  12-month rolling = [2025-01〜2025-12] / [2025-06〜2026-05] (2 fold)
  事前予想: dm_rank_1_3 は (i) で 76.5% で 2026P 単独崩壊が観察された。
            (iii) は順序だけなので結果同じ (sanity check)。
            (ii) で 2026 春が単独崩壊なら U1-e 棄却 (= fold 選択ではなく
            真の春崩壊)。崩壊が 2025-04〜2026-04 にまたがれば季節性
  判定基準:
    - (i) と (ii) と (iv) すべてで「直近 fold 単独崩壊」が現れる:
      fold selection bias = 強く棄却 (= 崩壊は本物)
    - (ii) または (iv) でパターン消失 (全 fold 平準): bias = 強く支持
    - 一部分割で消失: bias = 弱く支持

N2 (同オッズ帯 LGBM vs MING):
  入力: scripts/dump_predictions.py の per-pick CSV
  計算:
    (a) 各 fold で MING dm_rank_1_3 picks の odds 分布: median, 25%, 75%
    (b) その odds 帯 (= MING の IQR 範囲) で LGBM 戦略 (kelly_ge_05, ev_ge_105,
        prob_ge_20) を絞り、bets / hit_rate / return_rate / CI 下限を計算
    (c) MING と LGBM (同オッズ帯) の min_lo 比較
  事前予想: MING dm_rank_1_3 の odds median は ~3-5 倍 (favorite 寄り)。
            同オッズ帯の LGBM 戦略は bets が大幅減 (MING 同等水準で kelly
            条件も pass する pick は少ない)。
  判定基準:
    - 同オッズ帯 LGBM の min_lo が MING ± 5pp 以内: U2-(d) 弱く支持 (class effect)
    - 同オッズ帯 LGBM の min_lo が MING より 5-15pp 低い: border zone
    - 同オッズ帯 LGBM の min_lo が MING より 15pp 以上低い: U2-(d) 弱く棄却
      (= MING に LGBM が持っていない情報源)
    - 同オッズ帯 LGBM の bets が < 30 (fold あたり): 判定不能
