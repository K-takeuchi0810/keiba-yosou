§6 事前固定 rubric の data 適合性 sanity check (v4 §4 事故 3 予防)
-----------------------------------------------------------------

本 plan で採用する判定基準:
  1. Phase B1 採用 = 4 条件 AND (val Brier, reliability gap, robust=Y >= 1, holdout >= 80%)
  2. B1-S0 → B1-S1 gate = H3-3 で「Tier 2/3 features のうち N (>=1) 個が
     2026Q1 LGBM 崩壊の causal explanation 候補」と特定
  3. シナリオ A/B/C 分岐 = val Brier 改善 + reliability gap + filter_sweep の合成

  Data 適合性 check:
    - 条件 1 の「val Brier ≤ 0.055」は 2025 val data で計算可能、適合 OK
    - 条件 1 の「holdout >= 80%」は 2026 Q3-onwards で計算可能、ただし時点で
      ~4 ヶ月後のため fresh data 蓄積を待つ必要、適合 PARTIAL
    - 条件 2 の「causal」は 2026Q1 small sample (3 ヶ月) で identifiability の
      問題あり。「causal 候補」「妥当な correlation」程度に弱める必要、適合 PARTIAL
    - 条件 3 の合成は閾値依存、§0 rubric の controlled-ness × dataset 限定性で
      書き分け予定、適合 OK

  事前 rubric が data で支持されないリスクの記録:
    - 条件 1 の Brier 0.055 は v5 の 0.0606 と比較した 9% 改善目標。
      本来 Brier 連続値で feature importance を見るべきところ、binary 採否で
      切るのは selection bias の risk。Plan §採用判定基準でこの risk を明記する
    - 条件 2 の N=1 (Tier 2/3 features のうち 1 個 causal) は data scarcity への
      譲歩、過小要件のリスク。Plan §採用判定基準で N>=2 に上げる選択肢併載

§B1-S0.3 対立仮説 3 つ (= 事前予想を覆す可能性)
-----------------------------------------------

H_alt_1: 「Gate-2 (a) は実は p >> 0.05 (= 非有意)。
          Welch t-test を月次 Brier 5 ペアに適用すると、Brier の本来的
          variability (= 月内 sample 由来) が dominant で、季節 effect は
          埋もれる。+3.3σ は SD 算出が 2025 月次 std (=0.0121) を使っているが、
          これは year-aggregated std で、month-level の within-month variation
          を反映していない可能性」
   反論: bootstrap CI を併走するので、p 値が border zone でも CI 情報で
         判定可能。Gate-2 (a) failure 自体は plan v2 §7.4 で想定済 (C) pivot 経路
   含意: Step 2 で p >= 0.10 だった場合、即 §7.4 へ pivot を準備しておく

H_alt_2: 「Step 3 H3-3 で causal candidate が 「>= 2σ anomaly 軸」を 5 軸全部で
          出さない (= candidate ゼロ)。2026Q1 LGBM 崩壊は 5 軸のいずれにも
          局在しない複合効果 (= odds shift + features shift + market regime
          shift の同時発生) で、軸分解では検出不可」
   反論: 軸分解 = 必要条件であり十分条件ではない。複合効果でも 1 軸では
         弱い signal が出るはず。candidate ゼロは「分解粒度が粗い」を意味し、
         step 3 を「軸 + 軸 interaction」拡張する選択肢を保留しておく
   含意: H3-3 で candidate ゼロなら、step 3 を 2-3h 延長して軸 interaction
         (= 馬場 × 距離 等) を 1 段試す前に Gate-2 (c) failure 受け入れ判断

H_alt_3: 「Step 4 SQL audit で 9 features 全部 audit clear 通過し、Step 5 で
          Gate-2 ALL Y。しかし B1-S1 着手後に SHAP analysis で 「Tier 2/3
          features の効きはほぼゼロ」と判明。Gate-2 通過 = false positive」
