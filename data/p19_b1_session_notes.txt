§1 事前予想 (= v4 evidence 再読前の私の判断)
----------------------------------------------

問い (i): Phase B1 (LGBM v6 + Tier 2/3 features, 推定 25-30h) で利益化する確率は何 %か?

  事前予想 (証拠を再読しない状態):
    - P(val Brier ≤ 0.055) = 0.55-0.70
        理由: 4 年 TRAIN + Tier 2/3 で Brier の数値改善は base rate で期待される
    - P(reliability gap [0.3, 0.4) bin が 0.10+ に回復) = 0.40-0.55
        理由: Isotonic 校正 + race 内 Σ=1 の二重正規化が calibrator 層では治らない
              構造問題 = features 追加だけでは不十分の可能性大
    - P(filter_sweep で robust=Y >= 1) = 0.20-0.35
        理由: H1 N7 で cliff at R=80 観察 (H2 N9 で R=76 に修正)、73 戦略すべて未達
              の現状から、val Brier 改善が即 robust=Y に転化する保証ない
    - P(PRODUCTION holdout 2026 で絞り運用 >= 80%) = 0.20-0.30
        理由: 上記 3 つの AND 条件 + 2026 hold-out 期間が小 (~5 ヶ月)
              + Brier 改善が return_rate 改善に転化する transmission link 未検証

  最終確率 (4 条件すべて満たして「利益化」と定義した場合): 0.15-0.25
  「val Brier 改善だけ」を利益化のサインと取れば: 0.55-0.70

  証拠ベース見積もり可能か:
    - Yes (partially): v4 N1 月次 Brier 表 + reliability bin gap + N8 quarterly
                       return_rate の組合せで、Brier vs return_rate transmission link
                       の弱さを定量化可能 (= 2026Q1: Brier 良好 0.060-0.081, return 41-42%、
                       2026Q2: Brier 悪化 0.117-0.125, return 84-91%)
    - No (partially): Tier 2/3 features が新規追加された場合の val Brier 改善幅は
                      実走前には base rate しか出せない (feature engineering 効果は
                      実走で初めて分かる)

問い (ii): 上げる最短経路は B1 か、別経路か?

  事前予想 (証拠を再読しない状態):
    候補:
      (A) Phase B1 (LGBM v6 + Tier 2/3, 25-30h): P_profit 0.15-0.25
      (B) calibration-only fix (Isotonic refit + Σ=1 規制, ~5h): P_profit 0.10-0.20
      (C) operational hedge (Q-aware split: Q1 は MING, Q2-4 は LGBM, ~3h):
          P_profit_change unclear (= 分散化、平均は変わらない可能性)
      (D) 別 data source (パドック・血統 web API, 10-15h): P_profit 0.15-0.30、
          ただし開発期間中 90 日のオッズ drift で評価不能化リスク
      (E) v5 継続 + データ蓄積 (0h、6-12 ヶ月待ち): P_profit 0.10-0.20
          (現状運用が継続的に -1% 程度回ると仮定)
      (F) 採用判定基準を 80% → 76% に下げる (~1h): P_profit 0.20-0.30、
          ただし基準を下げるだけで実体は改善しない (= cosmetic)
      (G) 新 model architecture (CatBoost, neural ranking, 30-40h):
          P_profit 0.15-0.25、開発期間最長

    最短経路の解釈:
      a. 「期待利益化確率 / 時間」での最短: (C) operational hedge or (B) calibration fix
      b. 「最大利益化確率」での最短: (A) Phase B1 + (D) 別 data source の組合せ
      c. 「リスク minimum」での最短: (E) v5 継続

    私の事前推奨: (A) Phase B1 を採用するが、scope を絞った「B1-lite」に変更:
      - Tier 2/3 を full 7 features ではなく "selected 3 features" に絞る (= H3-3
        root cause 探索を 2-3h 実行 → 結果に基づき 3 features 選定)
      - LambdaRank は採用、ただし binary CE と A/B 並列訓練
      - 4 年 TRAIN は採用、ただし 2024 のみ追加 (= 3 → 4 年、5 年は cost 比メリット薄)
      所要: 15-20h、P_profit 0.20-0.30 (full B1 より低いが時間効率が良い)

問い (iii): Phase B1 scope を v6 再訓練のまま維持か、再定義か?

§6 事前固定 rubric の data 適合性 sanity check (v4 §4 事故 3 予防)
-----------------------------------------------------------------

本 plan で採用する判定基準:
  1. Phase B1 採用 = 4 条件 AND (val Brier, reliability gap, robust=Y >= 1, holdout >= 80%)
  2. B1-S0 → B1-S1 gate = H3-3 で「Tier 2/3 features のうち N (>=1) 個が
     2026Q1 LGBM 崩壊の causal explanation 候補」と特定
  3. シナリオ A/B/C 分岐 = val Brier 改善 + reliability gap + filter_sweep の合成

  Data 適合性 check:
    - 条件 1 の「val Brier ≤ 0.055」は 2025 val data で計算可能、適合 OK
    - 条件 1 の「holdout >= 80%」は 2026 Q3-onwards で計算可能、ただし時点で
      ~4 ヶ月後のため fresh data 蓄積を待つ必要、適合 PARTIAL
    - 条件 2 の「causal」は 2026Q1 small sample (3 ヶ月) で identifiability の
      問題あり。「causal 候補」「妥当な correlation」程度に弱める必要、適合 PARTIAL
    - 条件 3 の合成は閾値依存、§0 rubric の controlled-ness × dataset 限定性で
      書き分け予定、適合 OK

  事前 rubric が data で支持されないリスクの記録:
    - 条件 1 の Brier 0.055 は v5 の 0.0606 と比較した 9% 改善目標。
      本来 Brier 連続値で feature importance を見るべきところ、binary 採否で
      切るのは selection bias の risk。Plan §採用判定基準でこの risk を明記する
    - 条件 2 の N=1 (Tier 2/3 features のうち 1 個 causal) は data scarcity への
      譲歩、過小要件のリスク。Plan §採用判定基準で N>=2 に上げる選択肢併載
