## 項目別

- **回収率 (本丸): 4/5 (前回 4/5, ±0)** — 89.0% は控除率 +9pt 超え、control +28.2pt の優位は本物。ただし +100% (収支プラス) には届かず -11%、サンプル 105 戦は標本としてはまだ薄め。前回の 84.9% は期間バイアスを含む幻と判明したが、新 89.0% は walk-forward + sweep で両期間 80%+ という再現性を確認済のため数値の信頼度は前回より高い。±0 だが質的には改善。
- **EV 計算の整合性: 2/5 (前回 2/5, ±0)** — `_investment_probability` の blend/discount 構造、calibrator との二重がけ問題は未着手。フィルタが「校正後 EV」依存から「重賞+場+人気帯+信頼度除外」のヒューリスティック依存に実質シフトしたため、EV 計算の歪みは収益性に対しては一時的に bypass されているが構造課題は残置。
- **Kelly fraction / 投資割合: 2/5 (前回 2/5, ±0)** — `min(kelly, 0.05)` 表示のみで未着手。105 戦 / -11% で「均等ベットしてもまだマイナス」の状況なので、Kelly を本気で使うのは +100% 超え後で良い。優先度は低いまま。
- **買い目フィルタの実用性: 4/5 (前回 3/5, +1)** — **buy_only=0 → 105 戦** の解除は本専門家の最重要項目。`wl_ex_unsure_pop_1_4` (重賞+中山+京都+1-4 人気+信頼度除外) で 105 戦 / 89.0% は採用件数・回収率ともに合格圏。sweep で 6 種類の両期間 80%+ filter を検出済、`wl_odds_8_20` (100%+) も将来候補として控えており、フィルタ探索の枠組み自体が育った。+100% 未達のため 5 ではなく 4。

## 主な改善提案 (優先 1 件、最大 3 件)

1. **`wl_odds_8_20` を併用 filter として A/B 化** — sweep で 100%+ を検出した別 robust filter を `web/generator.py` の二系統 buy_only として並列出力し、`wl_ex_unsure_pop_1_4` (89.0% / 105 戦) と `wl_odds_8_20` (100%+) の合議 / 和集合での実弾運用を検証。+100% 超えへの最短経路。項目 1: 4 → 5、項目 4: 4 → 5 の同時引き上げを狙える。
2. **Kelly fraction を実ベット額に接続** — 現状の `min(kelly, 0.05)` 表示を `bet_size` カラム化し、105 戦 / 89.0% に対して均等 100 円ベット時の収支 (-1,150 円) と Kelly 加重時の収支差を backtest に出力。負け局面での分散効果を可視化。項目 3: 2 → 3。
3. **walk-forward レポートを scorecard に常時添付** — DESIGN 74.3% と EVAL 89.0% の差分が「期間バイアスではなく filter の頑健性」だと示すため、`data/backtest/` に `_design.json` / `_eval.json` の対を必ず残し、専門家レビューが両者を自動比較できるようにする。前回の 84.9% 幻のような事故を防ぐ仕組み。
