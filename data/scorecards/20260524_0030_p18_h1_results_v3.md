## 2. 棄却された仮説 (= データが支持しない、と現時点で言える)

### v3 で新たに棄却される仮説 (= H1 直接結果から、強さは「弱く棄却」)

**仮説 R3 (新規)**: 「判定基準 (0.80, 0.50) は researcher degrees of freedom の典型で、現データに「丁度通らない」位置にある」

* **証拠**: N7 で 6 閾値ペアにわたり Y 件数の隣接ジャンプ最大 1。事前固定した急変基準 (5 件以上) 未満。現状閾値の周辺 ±5pp で robust=Y 件数が動かない。
* **棄却強度**: **弱く棄却**。隣接ジャンプ基準 1 つだけで判定しており、別の感度尺度 (例: 戦略単体での閾値変動の積分) で見たら違う結論になる可能性はある。
* **残課題**: R% 軸では (75, *) → (80, *) で point_robust 自体が落ちる cliff edge は観察された。これは selection bias の証拠ではない (80% = JRA 控除率超え という外的根拠あり) が、「cliff edge にあるという観察」は記録する。

### v2 から維持される棄却

**R1**: 「2026 春に payout 分布レベルの regime shift」(oracle 自身が落ちていない → 棄却)
* v3 N3 は「予測者の return_rate」ではなく「oracle (勝率 100%)」での比較。同季節 +63 pp は **両側 p=0.198** で棄却基準 (p<0.05) 未達 → R1 棄却の補強。ただし片側 0.099 で「border zone」、強く棄却ではない。

## 3. 未確定 (= 現データで判定できない、要追加検証)

### v2 から進展した U3 (= R2 経由で v2 U3 へ降格、v3 で N7 経由で部分解決)

**U3 v2**: 採用判定基準「点推定 ≥ 80% かつ CI 下限 ≥ 50%」が現実的に achievable か。
**U3 v3 の進展**: N7 で **U3-(d) (判定基準そのものの selection bias)** は弱く棄却。これで U3 の他の対立仮説 (a) 真の return_rate が 0.80 に届いていない、(b) bets 不足、(c) i.i.d. 前提崩れ、に絞られた。N6 と組み合わせると:

* U3-(a): N6 で favorite 戦略 (dm/tm rank_1_3) は point return_rate も 75-83% で **0.80 に近接して達成困難**、long-shot は < 0.50 で fundamentally insufficient → (a) 部分採用
* U3-(b): N6 favorite req_n=283-318 は observed bets=355-377 と同等 = bets 不足ではない。長 shot/middle 戦略は req_n が大幅不足。**class 別で異なる**
* U3-(c) i.i.d. 前提崩れ: 未検証 (H2 N1/N8 で扱える)

**v3 後の U3 強度**: 「判定基準は妥当 (弱く)」「point return ≥ 0.80 部分は favorite class で borderline、long-shot で fundamentally 失敗」「bets 不足は class 別」。

### v2 から進展した U4 (= N3 で部分解決、ただし border zone)

* **観察**: 同季節 +63 pp、両側 p=0.198 (棄却基準未達)、片側 p=0.099、percentile 90.5%
* **棄却強度**: **弱く棄却**。片側 0.099 は 0.05-0.10 の grey zone で、N=10 (5+5) の permutation は power 不足
* **代替仮説の残置**:
  - U4 棄却 (R1 補強): 自然変動の範囲
  - U4 「border」: 真の差はあるが N=10 で確定不能
  - 隠れた regime shift (上側に寄っている事実は否定できない)

### v2 から進展した U6 (= N6 で class 別 verdict)

* **favorite class** (dm/tm rank_1_3): eff_std = full-pop の 8-24%、req_n=283-318 ≈ observed → **U6-(a) achievable** 弱く採用
* **long-shot class** (kelly_ge_05): point return 40.5% < 50% → **U6-(b) class 別 unreach** 採用 (= 構造的不可能)
* **middle class** (ev_ge_105, all): req_n が観察の 8-50 倍 → **U6-(c) 大量 bets 必要** で実運用では達成不能

**重要**: U6 が favorite で achievable でも、robust=Y 達成には **R% 80 部分も必要**で、これは N6 では解いていない。N6 単独で「Phase B1 OK」と読まないこと。

### v2 U5 / U1 / U2 は v3 範囲外 (H2 で扱う)

* U5 (Phase B1 で robust=Y 出るか): 未着手
* U1 (2026P fold underperformance 原因 a-e): H2 N1+N8 で扱う
* U2 (MING > LGBM が favorite class 効果か): H2 N2 で扱う

### U7 (v3 新規): 「N7 で観察された R%=80 cliff edge」が真に「point 性能の問題」か「閾値設計の問題」か

* **観察**: (75, *) で Y=1+hold=2、(80, *) で Y=0+hold=0。L% 軸は無関係。
* **対立仮説**:
  - (a) point 性能の構造的問題: 73 戦略中 1 つも 3-fold all-≥80% を満たさない = 真の能力不足
  - (b) 80% という閾値が偶然 cliff edge: 79% にしたら 5 戦略くらい出るかも (= 戦略性能分布が 70-80% に集中している)
  - (c) JRA 控除率超え (80%) 自体が ambitious すぎる業界水準: 控除率 75% (= 単複より高い券種) を採るのが現実的かも
* **再評価条件**: R% 軸をより細かく振る (例: 78, 79, 80, 81, 82) + 3 fold 別に point return 分布を視覚化

## 5. Phase B1 plan 着手判定 (handoff §「Phase B1 plan に進む条件」)

| 条件 | 状態 | 判定 |
|---|---|---|
| (1) Phase H1 完了 (U3, U4, U6 確定) | ✓ N7, N3, N6 でそれぞれ partial 確定 | **clear** |
| (2) 判定基準 (0.80, 0.50) が現実的 | 弱く clear (隣接ジャンプ max=1) | **weakly clear** |
| (3) 規範外バイアス (fold 選択、season selection) が H1 で否定 | season は弱く棄却、fold は H2 N8 未実施 | **partially clear** |

**handoff §74 の解釈**: 「fold 選択」と「season selection」が並列で書かれているが、N8 (fold) は handoff §H2 に分類されている。本セッションでは:

- **厳密解釈** (a): 「H1 で否定」= H1 範囲内で扱った bias の否定のみ → season selection は弱く clear、fold selection は H1 範囲外 → **条件 3 は (a) 解釈なら weakly clear**
- **包括解釈** (b): 「fold + season すべて否定」が必須 → fold は未検証 → **条件 3 は (b) 解釈なら not clear**

**私の判断**: handoff 自体に矛盾 (§74 と §H2 で fold の所属が違う) があるので、(a) 解釈を採るが、その判断を 1 文記録 + Phase B1 plan 着手前に「fold selection bias 検証 (N8) は plan に含める」と注記する。

**3 条件の総合**: **weakly clear / not strongly clear**

→ **Phase B1 plan 書き直しに着手するか** の決定は **§6 Step B (クロージング判断) でユーザに委ねる**。本セッションで plan を書き直すと「H1 終わり頃の判断強度低下リスク」と「week clearance での前進」のトレードオフ。handoff §H1 完了後 Step B は (i) H2 続行 (ii) クロージング (iii) Phase B1 plan 直行 の 3 択を示している。私の初期推奨は **(ii) クロージング**: H1 が weakly clear で B1 plan 書き直しは「強く clear」状態で始める方が結果的に効率が良い、と P18 セッション末の判断と整合する。

## 7. v2 → v3 で改善した点 / v3 残課題

### 改善した点

- §1 の 3 表 (N7 / N3 / N6) は実数値 (CSV + log) ベース、再現可能
- §3 の各 U に対し「v3 で進展した部分」と「H2 範囲」を明示
- §4 メタで「予想 vs 結果」3 件比較を表に出し、確証バイアス回避の効き目を可視化
- §5 で handoff の矛盾 (§74 vs §H2) を明示してから判断
- 4 回訂正パターンの再発なし (事前固定基準を機械適用)

### v3 で残る可能性のある瑕疵

- §3 U6 の class 別 verdict は「favorite/long-shot/middle」の 3 分類だが、実際の戦略空間はもっと連続的。3 分類が selection bias を持つ可能性
- §3 U4 の「border zone」判定は N=10 (5+5) の power 不足を理由としているが、power 計算自体は実施していない
- §5 の条件 3 解釈 (a) 採用は handoff の矛盾を「私の判断で解決した」もので、ユーザの本来意図と一致しない可能性
- §4 で「過剰な慎重さに振れていないか」を留保したが、判定する具体的方法を提示していない
- N7 で観察した R% 80 cliff edge を「外的根拠あり (JRA 控除率超え)」と片付けたが、JRA 控除率は単勝 20%、複勝 20% で他券種 (馬連 22.5%、3 連単 27.5%) は異なる。「80% で勝つ」の意味付けは券種依存で、現状の単勝固定でしか成立しない。Phase B1 で他券種を扱うなら閾値も連動するべき

**v3 も次回再読時に「やはり何かが混入していた」と訂正される前提**で書かれている。
