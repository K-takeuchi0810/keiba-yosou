## 主な改善提案 (優先 1 件、最大 3 件)

1. **`weights.json` の sweep を即実施** — リファクタで動かしやすくなった (EV 閾値 / Odds 下限上限 / Value 閾値) を `scripts/filter_sweep.py` で whitelist_only 326 戦に対して格子探索。「採用件数 ≥ 50 / 回収率 ≥ 90%」の点を 1 つでも見つければ buy_only=0 問題が解け、項目 4 を 3 → 4、項目 1 を 4 → 5 に動かす最短経路。リファクタの真価を即数値化する。
2. **`weights.json` のスキーマ・既定値を README に明記** — 重みが JSON 化された以上、どのキーが何に効くかをユーザ (= 自分) が後から追えないと将来事故る。`config.py` か `weights.json` に隣接する形でキー一覧と既定値・許容範囲を残す。期待効果: 提案 1 の sweep を安全に回せるようになり、将来の信頼度ラベル再定義 (前回提案 2) も同じ仕組みに乗せられる。
3. **動作不変テストを `pytest` に固定化** — 「P1-1 で backtest 数値が完全一致した」という事実を、今後のリファクタで再現可能にするため、`scripts/backtest.py` の主要 KPI (races_bet, return_rate, whitelist_only_*) をスナップショット化したテストを `tests/` 配下に追加。期待効果: 今後 weights.json をいじったとき「意図せず数値が動いた」を即検出できる = 提案 1 の sweep を回す前提条件。
