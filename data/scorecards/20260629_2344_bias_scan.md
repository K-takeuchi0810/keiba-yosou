## 横断的に見た優先課題 → 本セッションで対応した修正

7 名のうち 3 名 (validation/profitability/prediction-logic) が **多重比較と subject=all の有効 n** に収束。以下を `bias_scan.py` に反映済 (本 scorecard と同コミット):

1. **subject=all のレース内相関補正** (validation 最重要) — `Cell.n_races` を追加し有効 n=レース数を計上。all モードでは (a) gap の有意判定を None にし、(b) bias_severity を sqrt(n_races) で計算、(c) return 系指標を抑制 (全馬買いは戦略でなく馬単位 bootstrap も無効)。ヘッダに相関警告を明示。
2. **多重比較警告** (profitability/prediction-logic) — `n_cells_tested` を算出し MOST BIASED ヘッダに「約N セル×α=0.05 → 期待偽陽性 ~M」と「重み変更は 2024/2025 独立再現+holdout 通過後のみ」を表示。
3. **n_trusted / n_races / avg_field をコンソール表示** (profitability/prediction-logic) — セル行を `n=…(nr=… tr=…) fld=…` に拡張し、2024/2025 の return 基盤差と出走頭数依存を可視化。
4. **calibration_in_sample を stdout 表示** (profitability)。
5. **track×kaiji クロス追加** (prediction-logic)。
6. **dead import 除去** (`weather_name`/`ground_name`) + 障害フォールバックの "0"=truthy バグ修正 (code-quality, テストで検出)。
7. **tests/test_bias_scan.py 新設** (code-quality) — 10 ケース、境界値固定。
8. **原子書き込み (tmp+rename) + `--save` 無し警告 + n_odds_untrusted 記録** (data-pipeline/validation)。
