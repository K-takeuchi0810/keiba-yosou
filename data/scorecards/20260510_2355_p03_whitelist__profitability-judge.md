# 収益性 / 投資判断専門家 採点

## 総合: 3.0 / 5

P0-3 で `config.BUY_FILTER_DEFAULT` に `whitelist_mode/grades/tracks` を追加 (既定 ON)、`_is_buy_candidate` / `web/generator.py` / `scripts/backtest.py:_matches_buy_filter` の三箇所で AND 条件を適用、backtest に `whitelist_only_*` を追加。最新 backtest `data/backtest/20260510_235222_tan_p03-whitelist-with-only-all.json` (1,164 戦) で **whitelist_only (重賞 + 中山 + 京都ベタ買い、326 戦) の回収率 84.9%** を確認 = **JRA 単勝控除率 80% を初めて超えた**。前回まで「3 未満ルール」で 1.x〜2.0 を行き来していたが、controlling な単一改修で控除率を超える数値が出た以上、ルールに従い総合 3 を解禁する。**ただし** buy_only (whitelist + EV/Odds/Value) は依然 **0 件**、信頼度ラベルは依然順序逆転、Kelly も未着手で、4 以上は出せない。

## 項目別

- **回収率 (本丸): 4/5 (前回 1/5, +3)** — whitelist_only 326 戦で **的中率 25.8% / 回収率 84.9%**。控除率 80% に **+4.9pt 上乗せ**、全体ベタ買い 60.8% に対して **+24.1pt**。前回「唯一の光明」と呼んだ重賞 115.4% / 中山 91.4% / 京都 80.3% を **AND ではなく OR で束ねた合成セグメント** が、サンプル 326 戦 (= 統計的にも当てになる規模) で控除率を超えたのは大きい。実弾 +5% 相当の収益見込みが現実線。5 にしないのは (a) 84.9% は **控除率超え** であって +20% プラス収支ではない、(b) `_is_buy_candidate` 経由 (whitelist + EV/Odds/Value) では依然 0 件採用 = 「実 GUI で買い目に出る件数」は別途課題、(c) 単勝のみで複勝・馬連ベース未検証、の三点。
- **EV 計算の整合性: 2/5 (前回 2/5, ±0)** — P0-3 は EV 計算には触らずホワイトリストの AND 条件のみ追加。Brier 0.0575 / LogLoss 0.2098 (n=16,550) は前回と同じ確率精度。「校正済み確率 × 実オッズ」が市場に勝てない構造は不変。EV 周りは未着手なので前回維持。
- **Kelly fraction / 投資割合: 2/5 (前回 2/5, ±0)** — `min(kelly, 0.05)` 表示のみ問題は不変。維持。
- **買い目フィルタの実用性: 3/5 (前回 1/5, +2)** — `buy_only_bets = 0/1164` は **据え置き** (= EV/Odds/Value の三条件で whitelist と AND を取ると依然ゼロ件) なので「フィルタ機能死亡」の根本問題はまだ残る。**ただし** P0-3 で `BUY_FILTER_DEFAULT` の whitelist_mode が既定 ON になり、**whitelist 単独** ベタ買いが 326 件採用 / 25.8% 的中で運用可能ラインに乗った。「採用件数の絶対不足が解消された」点は前回 1 から +2 の根拠として十分。4 以上にはしない理由 = buy_only の 0 件は EV / Odds 帯のチューニングが別途必要、`relaxation` チェーンの実発動も未確認。
- **校正済み確率の信頼性: 2/5 (前回 2/5, ±0)** — by_confidence は今回も **接戦 68.3% > 高信頼 59.8%** で順序逆転継続 (高信頼 91 戦 28.6% 的中 / 59.8% 回収)。P0-3 は信頼度ラベル定義に手を入れていないので不変。維持。

## 主な改善提案 (優先 1 件、最大 3 件)

1. **`buy_only_bets = 0` を解消する EV / Odds 感度分析と既定値更新** — `scripts/filter_sweep.py` を whitelist_only 326 戦に対してのみ走らせ、`(EV閾値, Odds下限, Odds上限)` の格子で「採用件数 ≥ 50 / 回収率 ≥ 90%」の点を探索。326 戦のうち EV ≥ 1.05 が何件あるか、Odds 10-20 制約が何件削っているか分解する。期待効果: whitelist 内サブセットで控除率 100% 超え (= 真の黒字) が見つかれば、項目 1 を 4 → 5 に、項目 4 を 3 → 4 に押し上げる経路。これだけ単一の sweep でも GUI 実弾に直結する数値が出る。
2. **信頼度ラベルを whitelist 内データで再定義** — 現行の高信頼 91 戦 / 59.8% 回収は使い物にならないが、whitelist_only 326 戦 / 84.9% は使える。「whitelist 内 + calibrated_prob ≥ 0.25」を新「高信頼」に固定、whitelist 外は全部「接戦」に降格、のような **後ろ向き backtest 由来の経験則ラベル** に置き換える。期待効果: GUI 上の信頼度バッジが投資判断材料になる。項目 5 を 2 → 4 に動かせる。
3. **whitelist_only を複勝・馬連 BOX でも測定** — 現状 84.9% は単勝のみ。重賞は荒れやすいので **複勝の方が回収率が安定** する可能性が高く、もしそうなら「重賞 G1-G3 の複勝ベタ買い」が控除率超え + 分散低減で実弾運用の本命になる。`scripts/backtest.py` を `--bet-type fuku` / `--bet-type uren-box` で同一 326 戦に走らせて回収率を比較。期待効果: 84.9% を超える戦略が見つかれば項目 1 を 4 → 5 に確定できる。

## 前回からの差分 (2.0 → 3.0, +1.0)

- 回収率: 1 → 4 (+3) — **whitelist_only 326 戦 / 84.9% で控除率初超え**。326 戦は統計的にも信用に足るサンプル、+24.1pt の合成効果は構造的勝利
- EV 計算の整合性: 2 → 2 (±0) — P0-3 は EV ロジック未変更、不変
- Kelly fraction: 2 → 2 (±0) — 未着手、維持
- 買い目フィルタの実用性: 1 → 3 (+2) — `buy_only=0` は据え置きだが whitelist 単独で 326 件 / 25.8% 的中の **運用可能フィルタが初めて成立**。採用件数ゼロ問題は半分解消
- 校正済み確率の信頼性: 2 → 2 (±0) — 信頼度ラベル順序逆転継続、P0-3 は触っていない

**総括**: P0-3 は前回 scorecard の優先提案 1 (重賞 + 中山京都ホワイトリスト) を **そのまま実装し、backtest で控除率超えを取り切った** 点で、過去の改修群 (P0-1 校正導入 / P0-2 min_count / P1-3 logging) の中で **初めて回収率の数字を動かした改修**。これまで「正しい方向だが届かない」だった改善ラインが、ようやく「実弾投入で +5% 程度のプラスが見える」位置に到達した。次の一手は (a) whitelist 内での EV/Odds 二段絞り、(b) 信頼度ラベル再定義、(c) 複勝・馬連での再測定、の三方向で 4 点台を狙う段階。
