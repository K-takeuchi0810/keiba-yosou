### 5.2 シナリオ別判定 (v2.1 で OP-1 = multi-fold row を新規追加, P9 修正)

| シナリオ | 条件状態 | 判定強度 | アクション |
|---|---|---|---|
| **A** | 1+2+3+4 ALL | 「強く支持」(B1 利益化) | Phase B2 |
| **A-partial** | 1+2+3 (条件 4 待ち) | 「弱く支持」 | 採用、絞り運用で 2026 Q3 holdout 待ち |
| **B** | 1+2 のみ | 「弱く支持」 | calibration 層改善試行 |
| **C** | 1 のみ | 「border zone」 | 新 architecture 検討 |
| **D** | 1 NO | 「弱く棄却」 | scope 大幅 revision |
| **multi-fold (採否補助 row)** | quarterly fold (Q1-Q4) + rolling 12m + 季節別 の 3 scheme 中 >= 2 scheme で robust=Y or min_lo >= 75% | Gate-2 (b) 通過の必須条件 | この row 未達なら、シナリオ A の判定強度を「強く支持 → 弱く支持」に降格 (= A → A-partial 強制) |

**multi-fold row の運用** (v2.1 OP-1 で新規明文化、Gate-2 (b) 正面対応):
- 既存 row (A〜D) は **条件 1+2+3+4 の AND による採否判定**。multi-fold row は **fold scheme 分割への robustness 補助判定** で、A〜D の row と直交する次元
- B1-S5 で `scripts/filter_sweep.py` を quarterly fold (`--walk-forward-3fold`)、rolling 12m (= 自前で 12m window 集計)、季節別 (= Spring/Summer/Autumn/Winter 別 fold) の 3 scheme で実行
- 3 scheme 中 >= 2 scheme で robust=Y を達成、または robust=Y 未達でも min_lo >= 75% を達成、なら multi-fold row PASS
- multi-fold row PASS は **Gate-2 (b) 通過の正の必要条件**: B1-S0 完了時 (= 本セッション中) に「§5.2 に multi-fold row が組込済 + 評価方法が具体化」が evaluate される
- **採否への接続**: 1+2+3+4 ALL Y であっても multi-fold row 未達なら **シナリオ A → A-partial 強制降格**。すなわち 4 条件 binary を pass しても fold-scheme-fragility ありとして「絞り運用 + 2026 Q3 holdout 待ち」へ送る
- **filter_sweep 既存機能との対応**: `--walk-forward-3fold` (= quarterly に相当)、`--by-track-3fold` (= 季節別に近い)、rolling 12m は新規 scheme として B1-S5 で追加実装
- **採用判定 §5.1 連続改善 7 指標との連動**: 指標 D (recent-3fold で robust=Y にならない戦略の min_lo 改善幅) は multi-fold row と一部 overlap、ただし D は連続改善の grade、multi-fold row は binary の補助判定で意味が異なる
