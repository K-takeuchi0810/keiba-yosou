## 項目別

- **シグナル網羅性: 4/5** — シグナル層には一切触れていない。前回どおり維持。
- **重み妥当性 / 過適合リスク: 3/5** — `min_count=50` の構造は前回のまま。今回の改修は過適合経路を増減しない。`weights.json` magic number 直書き繰越も同じ。
- **信頼度判定 / 確率推定: 4.5/5** — `_apply_calibrator` (rules.py L760-767) で `calibrator.get("type") != "bin"` 時に `logger.warning("calibrator disabled: unsupported type=%r ...")` を 1 行追加。前回までは `type` が想定外の値だと **何の警告もなく raw 確率にフォールバック** していた (= calibrator.json を編集したつもりが効いていない事故が無音で起きる) のを潰した。コメント (L761-762) も「サイレント無効化を避け警告を 1 度出して raw 確率にフォールバック」と意図が明示されている。設計の素点は変わらないが、「観測可能性」軸では確実に良化。<br>満点でない理由: (a) `logger.warning` が **毎回呼び出しごとに発火** する実装で、predict ループが回ると同じ警告が大量出力される (前回スコアカードの優先 #2「初回呼び出し時のみ logger.info」の **once-only ガードと整合させていない**)、(b) `bins` が空 / 全 bin が空 / `min_count` 過大による「実質無効」のサイレント失敗モードは依然ノーガード (前回提案 #2 の核心は未解消)。素点 4.5 を割るほどではないので維持。
- **デッドコード / 設計の整合性: 3/5** — 前回優先課題の dead feature 4 件 / dead weight 2 件 / `_score_one` 491 行肥大 / magic number 50+ 箇所は **全て未着手**。今回の追加 1 行は `_apply_calibrator` 内に閉じており整合性は乱していないが、ロジックそのものの設計負債は据え置き。
- **本番運用との乖離リスク: 3/5** — むしろ良化方向 (誤設定が静かに無視される事故を 1 種潰した) だが、前述のとおり「全 bin が `min_count` 未満で恒等寄せされ実質 calibrator が消える」ケースは依然ログに出ない。総合的に素点維持。

## 主な改善提案 (優先 1 件、最大 3 件)

1. **前回優先 #2 と統合した一発ログ化** — 現状の `logger.warning(...)` は呼び出しごとに出るので、`_apply_calibrator` の **モジュールレベルで一度だけ** 「`type=bin` / 有効 bin=N / 恒等寄せ bin=M / 空 bin=K / `min_count`=50」を `logger.info` 1 行 + 異常時のみ `logger.warning` を流す形に統合。`_load_calibrator` のキャッシュキー (`mtime`) と同じタイミングで一度だけ集計すれば、ログ氾濫もサイレント無効化も同時に解決する。
2. **【繰越】`_score_one` 491 行の責務分割 + magic number 50+ 箇所の `weights.json` 移管 + dead feature/weight 削除** — 前回優先 #3 そのまま。今回の改修対象外。
3. **【繰越】`min_count` 決定根拠コメントと空 bin の除外** — 前回優先 #1 そのまま。今回の改修対象外。

## 補足

P1-3 は **前回スコアカードの優先 #2 (calibrator 適用統計のログ出力) を最小コストで先取りした改修** だが、対象を「`type` 異常時のみ」に絞ったため、本来狙うべき「全 bin が恒等寄せで実質無効」「空 bin だらけ」のサイレント失敗は依然検知できない。`logger.warning` が毎回呼ばれる点も、`_load_calibrator` のキャッシュ層と同居している以上 `mtime` 変化時のみログを出す once-only ガードが自然 (実装コスト < 5 行)。設計負債 (491 行関数 / dead feature / magic number) は全て据え置きで、観測可能性の小改善が他軸の悪化と相殺もしないため、総合 3.6 維持が妥当。次回は提案 #1 をマージしつつ繰越課題のいずれかに着手すれば素点 4 に届く。
