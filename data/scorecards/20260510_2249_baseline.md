## 横断的に見た優先課題 (Top 5)

7 名の改善提案を Claude (= 自分) が並べ替え、依存と効果で優先度付け。

### 🔴 P0-1. buy_filter の二重定義を統一する
- **指摘元**: 収益性 (#4)
- **問題**: `web/generator.py:31-34` の既定 `EV>=1.10 / Odds 2.0-8.0 / Value>=10.0` と `scripts/backtest.py` 既定 `EV>=1.05 / Odds 10-20 / Value>=0` が **完全別物**。「フィルタを掛けたら勝てるか」を測れる状態にすらなっていない。
- **修正**: `config.py` (or `predictor/__init__.py`) に `BUY_FILTER_DEFAULT` dict を 1 箇所定義し、`_is_buy_candidate` / `web/generator.py` / `scripts/backtest.py` / `gui/app.py` 全部が同じ定数を import。

### 🔴 P0-2. calibrator の少数 bin が過学習している
- **指摘元**: 収益性 (#4) + 検証プロセス (#7) + 予想ロジック (#3)
- **問題**: `calibrator.json` の bin 0.15-0.20 が count=27 で `calibrated_probability=0.3333` (raw 0.1725 の **約 2 倍**)。`shrinkage_alpha=30` でも吸収しきれず、偽の高 EV 候補を量産する温床。
- **修正**: `min_count=20→50` に引き上げ、未達 bin は隣接 bin と統合 or `calibrated = avg_probability` (恒等) に固定。`scripts/plot_calibration.py` で reliability curve を PNG 保存して可視確認。

### 🟠 P1-1. `_score_one` 491 行 + 直書きマジックナンバー 60 箇所
- **指摘元**: コード品質 (#6) + 予想ロジック (#3)
- **問題**: `predictor/rules.py:_score_one` (83-573) に同型 `if X_wins>=1: score += min(X_wins,N)*W` ブロックが 7 箇所重複、直書き定数が 60 箇所。`weights.json` 経由の 48 箇所と混在し可読性壊滅。`risk.graded_unproven_penalty` 等は dead weight (どこからも参照なし)。
- **修正**: 5〜7 個のシグナル関数 (`_score_track_type` 等) に分解 + 直書きを `weights.json` に移送 + dead feature 6 件 (`weight_trend` `recent_avg_starters` `same_day_leg_bias` `same_day_leg_samples` `same_track_type_runs` `estimated_leg_samples`) を削除。

### 🟠 P1-2. `_investment_probability` の三段がけを 2 段に減らす
- **指摘元**: 収益性 (#4) + 予想ロジック (#3)
- **問題**: calibrator 校正 → 信頼度別 blend → odds 帯 discount の三段で確率を歪める。`PRED_DISABLE_DISCOUNT=1` の比較 backtest が `data/backtest/` に保存されておらず、効果が追えない。
- **修正**: `--rule-version no-discount` で 5/2-5/3 と長期 (3,180 戦) の両方を取り、現行 vs 無効化の `return_rate` 差を保存。差が小さいなら discount を `weights.json` で 1.0 化、calibrator 単段で運用。

### 🟠 P1-3. 観測性 (`logging` ゼロ + サイレント例外 9 箇所)
- **指摘元**: コード品質 (#6)
- **問題**: `jvlink_client/client.py` の `except Exception: pass` が **9 箇所**。COM 失敗が握り潰されて GUI 上は無言で進行。`print(...)` が運用パスに 6 箇所残存、`import logging` ゼロ。原因不明のサイレント失敗が運用最大リスク。
- **修正**: `logger = logging.getLogger(__name__)` を導入、9 箇所を `logger.warning(..., exc_info=True)` に置換。`print()` 6 箇所も logger に寄せる。

### 🟡 P2-1. ダークモード時のハードコード色 + アクセシビリティ
- **指摘元**: モバイル HTML (#2) + GUI / UX (#1)
- **問題**: `index.html.j2` の `.buy-board` `.conf-tag` 等で固定白背景。`@media (prefers-color-scheme: dark)` 内で再定義されておらず、夜の iPhone で「白い箱に灰文字」WCAG AA 不通過。GUI も `--text-mute` on `--bg` が AA ぎりぎり、tap 領域 36px (44 未満)、`title=` `aria-label` ゼロ、`focus-visible` なし。
- **修正**: `--buy-bg` `--tag-bg` 等の変数化 + ダーク時再定義。GUI のタップ領域を 44px、`title=` を全ボタンに、サイドバーは `overflow-y: auto`。

### 🟡 P2-2. walk-forward / 期間分割評価が未整備
- **指摘元**: 検証プロセス (#7)
- **問題**: 直近 6 件 backtest 中 5 件が 20260502-20260503 集中、長期は 1 件のみ。`tuned-week-check` (52.2%) vs `current-week-check` (50.7%) は同期間 = 自己参照。walk-forward / CV 不在。
- **修正**: `scripts/walk_forward.py` を新設、`--window 30 --step 7` で学習窓と評価窓を分離して連続 backtest。

### 🟡 P2-3. `<details>` 開閉インジケータ非表示
- **指摘元**: モバイル HTML (#2)
- **問題**: `::-webkit-details-marker { display: none }` した上で代替が無く、「タップして開ける」ことが視覚的に伝わらない。
- **修正**: `summary.race-head::after { content: '▼'; transition: transform .2s }` を grid 3 列目に追加。

### 🟢 P3-1. ingested_files に mtime/size 保存
- **指摘元**: データパイプライン (#5)
- **問題**: `only_files` / `modified_since` を呼び忘れると週次 RACE 上書きを静かに見逃す。
- **修正**: `record_ingested_file` に mtime/size 列を追加、`is_file_ingested` を「ファイル名 + mtime 一致」判定に。

### 🟢 P3-2. `tests/` 不在
- **指摘元**: コード品質 (#6)
- **問題**: pytest 等の単体テストがゼロ。`predict_race` 等のリグレッション検出経路が無い。
- **修正**: `tests/` 新設 + `predictor/features.py` の純関数 5 本にスモークテスト。
